## 0. 本域与内核的关系（一句话）

> 小学学习 = 在 Jarvis 通用能力之上，**注入教育背景数据 + 挂载学习工具 + 施加年级与安全边界**。学生说「我不会这题」时，Jarvis **先自然答疑**，必要时才调用学习工具；说「今天不想做题」时，走共情与主动判断，**而不是硬拉回推题**。

## 0.3 持续性 / 主动：随开发顺手验（不单列切片）

> 结论：跨天学情记忆**已天然成立**（gap/画像/attempt 全落盘，新会话即带回；已核实退位漏洞跨会话仍在、队列自动转退位专练）；主动消息**机制已在跑**（`attempt_submit` 会吐"练后小结"）。故不专门留几天验，折进日常开发用下面三步顺手核：

- [ ] **续接**：新开 `hermes chat`，开场问无关的话，看它是否自然带回上次的退位漏洞 / 主动问"那道退位题练熟了吗"。
- [ ] **掌握晋升**：连续做对同类题 3 次（`required_streak: 3`），看 gap 是否转 mastered、不再被推。
- [ ] **主动不打扰**：确认主动消息只在合适时机出现，不刷屏（`proactive` 配置）。

## 0.6 本域执行框架：数据组件 + 业务操作（Agent 编排）

> 对应 L0 **P7（流程非硬编码）** 与 I2。详细模块/数据流见 [功能架构-模块与数据流](./功能架构-模块与数据流.md)（尤其 §15）。

本域**不规定固定流程**，只向 Agent 提供两类东西，由 Agent 据多模态输入 + 对话 + 上下文 + 当前学情**当场编排**：

| 类别 | 内容 |
|------|------|
| **数据组件（名词）** | 知识点目录 KP · 学情（知识点掌握档）· 题库 · 错因/补救（可选）· Wiki（可选）|
| **业务操作（动词，经 I2 挂工具）** | 讲解知识点 · 分析学情 · 选题/出题 · 判分（题库/freeform）· 拍照归类入学情 · 生成家长报告 · 定 20 分钟微计划 |

**编排示例（同一输入，意图不同 → 业务不同）**：

| 学生输入 | Agent 推理意图 | 编排操作 | **禁止** |
|----------|----------------|----------|----------|
| 「讲讲 / 学第一单元 / 什么是…」 | 讲新课 | 对话讲解（Wiki/KP 可选） | 出题、`push_queue_peek` |
| 「出几道题 / 练一练」 | 求练习 | `questions_suggest` → `question_get` → `attempt_submit` | 读离线队列 |
| 「退位还不会，再练」 | 补薄弱 | `questions_suggest(focus=remediation)` | 无关旧题 |
| 拍题 +「这道我不会」 | 求讲解 | 讲解知识点（必要时 Wiki 对齐）|
| 拍批改作业 +「记错题 / 复盘 / 看看错哪记一下」（语义，非固定口令） | 入学情/复盘 | 归类 → 判分 → 入学情 → 点评 |
| 拍题 +「再出两道像这样的」 | 求练习 | 归类 → `questions_suggest` 按 KP |

> **push_queue**：离线微计划缓存，**不作**默认练题入口（2026-06-24 纠偏）。即"拍照"本身不绑定任何后续业务；走哪条由 Agent 判断。**严禁把这些操作固化成"拍照=必走某管线"的状态机**（违反 P7）。
