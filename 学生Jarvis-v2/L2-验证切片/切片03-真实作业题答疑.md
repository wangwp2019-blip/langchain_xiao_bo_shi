# 切片03 · 真实作业题答疑（脱离种子题库）

> 一个切片 = 一条可证伪假设 + 一个真实用户的通过标准 + 最小验证动作。
> **本文件不得重定义 L0 能力契约。** 建立在 [切片01](切片01-二年级自然对话.md)（学伴感）+ [切片02](切片02-学习闭环自然融入.md)（学习闭环·证据优先）已验证的基线之上。

---

## 1. 假设（可证伪）

> **H3：** 当孩子拿来一道**不在题库里的真实作业题**并卡住时，Jarvis 能：
> （a）**自然答疑**——讲到二年级能懂、分步、正确；
> （b）**守住证据优先**——在没有题库判分证据时，**不臆测**「你这类题总错」，最多说「我们先看这一题」；
> （c）**学情可纳入**——这次"真实题卡住"能以某种方式被记下（哪怕暴露"需要新机制"）。
>
> 若脱离题库后 Jarvis 开始**乱判 / 凭空下结论 / 答疑质量明显下滑**，则当前接入方式对真实内容不成立 → 需要 LLM 答疑或判分能力（转为一次 build）。

## 2. 为什么先验这条（动机）

- 切片02 验的是**题库种子题**；但真实孩子用的是**作业本上的题**，不在库里、`attempt_submit` 无法判分。这条假设回答：**"真实内容"这一步，现有对话+证据机制能不能撑住，值不值得为它投入 OCR/判分。**
- 它验证 L0 的 **C-KNOW（答疑）/ C-GOV（不臆测）** 在"无结构化判分"条件下是否仍稳。
- 是**多模态拍照（C-IO 输入）的前置价值验证**：OCR 只是把"家长打字输入真题"自动化；先确认真实内容有价值，再决定建不建 OCR。

## 3. 通过标准（可观测、二元判定）

| # | 标准 | 测量方式 | 通过线 |
|---|------|----------|--------|
| 1 | **答疑质量** | 拿 2–3 道真实作业题（含一道孩子真卡住的），看讲解 | 讲解正确、适龄、分步；孩子能跟上 |
| 2 | **证据优先守住** | 无题库判分时，是否乱下结论 | 不臆测「你总错X」；要么只就本题讲，要么明说「先看这一题」 |
| 3 | **学情可纳入** | 这次卡住能否被记录/利用 | 能记下；**或**明确暴露"缺机制"（视为有效 finding）|
| 4 | **价值感** | 家长主观判断 | 「能做真实作业」明显比「只做种子题」更想用 |

## 4. 停止条件（R6）

- **成功即停**：命中标准 → 把「真实作业题答疑」沉淀进 L1，并据 finding **决定是否立项建 OCR/LLM 判分**。
- **证伪即转**：脱离题库就退化 → 明确"需要 LLM 答疑/判分能力"，**立为下一个 build 切片**，不动 L0。

## 5. 最小验证动作（R4/R5）

> 全程 WoZ，**不建 OCR、不写新解析**。真实题用"家长打字输入"模拟拍照识别的产物。

- [x] 复用已验证基线：`hermes chat` + agent-student（含新 `question_get`）+ 同一名孩子 `g2-stu-01`（顺带验跨会话续接）。
- [ ] WoZ：家长从孩子**真实作业本**挑 2–3 道题（含一道孩子真卡住的），打成文字发给 Jarvis。
- [ ] 重点盯：题不在库里时，Jarvis 是答疑（好）还是硬套题库/乱判（坏）；说薄弱点时有没有守住证据优先。
- [ ] 不做：不接 OCR、不接 LLM 判分、不扩题库。本切片就是为"要不要建这些"提供证据。

## 6. 真实用户与周期

- 用户：同一名真实二年级孩子 `g2-stu-01`（盈熙），延续已建立的记忆与退位漏洞。
- 周期：1 次会话（约 15 分钟），2–3 道真实作业题。
- 记录方式：每题记 ①答疑是否正确适龄 ②无判分时是否守证据优先 ③能否纳入学情/暴露缺什么机制 ④家长价值感一句话。

## 7. 结果（2026-06-21 回填）

- 结论：**✅ 通过（H3 核心成立）**——脱离题库做真实应用题，答疑质量与证据优先**均未退化**；标准3 以「明确暴露缺机制」形式达成（见下）。验证用真实作业题：①小明30/小红多24→30+24=54（孩子误用减法答6）②48−19=29（退位）。
- 证据：
  - **标准1 审题答疑 ✅**：题1 精准纠正「比…多→加法」并给口诀；题2 先讲「分给→减法」再列竖式，借位讲解全对得 29。分步、适龄。
  - **标准2 证据优先 ✅**：问「应用题是不是老错」时**未臆造**应用题薄弱，转而归因到**真实存在的退位漏洞**（有据）；`question_get` 正确取题面（F1 生效，未再翻文件）。
  - **标准3 学情纳入 → finding**：两道应用题 out-of-bank、未走 `attempt_submit`，**未纳入学情**（Jarvis 推练习仍只能用旧退位 gap + 题库题 q-g2m-005）。符合切片预期，视为有效 finding。
  - **续接彩蛋**：主动把 48−19 接上已有退位漏洞、从 gap_remediation 队列挑退位题巩固——§0.3「续接 / 主动引导练漏洞」顺带验过。
  - **标准4 价值感 ✅**：家长确认「能做真实作业」明显比「只做种子题库」更有价值、更想用。
- 关键决策（本切片最大产出）：
  - **不需紧急建 OCR**。真正缺口是「**真实作业题纳入学情**」的机制（即席建题 / LLM 判分）；OCR 只是其输入自动化，更靠后。
  - **F3（证据优先增强，可选）**：被问到「无证据领域」（如应用题）时，AnswerGate 宜要求明说「我还没有你这方面的练习记录」，而非转移到别的有据漏洞。
- 下一步：立一个 **build 切片**「真实作业题纳入学情（LLM 判分 / 即席建题）」，先验最小可行，再决定是否接 OCR。
