## 3. 一道题的标准流程（自然语言优先）

> ⚠️ **不要用命令喂答案。** 切片02 要验的就是「自然融合」，所以**默认全程自然语言**：孩子在 `hermes chat` 里直接和 Jarvis 对话，**Jarvis 自己出题、判分、记漏洞**（它会调用 `push_queue_peek` 取真实题、`attempt_submit` 判分——这些工具已注册给模型，系统规则也要求它必须查工具而非瞎猜）。

正常一轮长这样（**你只需观察，不用敲命令**）：

1. 孩子说「我们做两道题吧」之类 → Jarvis 从练习队列挑一道**真实题**（如 47+38）问孩子。
2. 孩子口头/口算说出答案（**答错也照说**）。
3. Jarvis 应当**自己调用 `attempt_submit`** 判分，再据此说话——对就鼓励；错就指出是「进位/退位」问题，且这个结论来自刚生成的 gap，不是套话。

你要做的只有两件事：
- **盯**：Jarvis 说的「你这里要加油」是不是引用了对话记忆 + 真实证据，而非凭空套话。
- **事后核对**（这是观测，不是输入答案）：

```bash
PY=/home/administrator/.hermes/hermes-agent/venv/bin/python
$PY -m agent_platform.learning.cli_student attempt list g2-stu-01 --limit 10   # 刚才那次作答是否记下、判分对不对
$PY -m agent_platform.learning.cli_student gap list g2-stu-01                    # 是否据此沉淀了漏洞（证据）
```

### 如果 Jarvis 没自动记录（fallback，且本身是一条发现）

若 `attempt list` 里没出现孩子刚才那次作答 → 说明 Jarvis 没去调 `attempt_submit`，**自然闭环没闭合**。这是一条重要结论，记到切片02 §7。为不打断验证，可临时手动补录这一题再继续观察：

```bash
$PY -m agent_platform.learning.cli_student attempt submit g2-stu-01 q-g2m-002 <孩子的答案>
```

### 参考题目（队列里的真实题，用于判对错 / fallback 补录）

| 题号 | 题目 | 正确答案 | 考点 |
|------|------|----------|------|
| q-g2m-001 | 23 + 45 | 68 | 不进位加 |
| q-g2m-002 | 47 + 38 | 85 | **进位加**（易错） |
| q-g2m-003 | 56 + 27 | 83 | 进位加 |
| q-g2m-004 | 68 − 35 | 33 | 不退位减 |
| q-g2m-005 | 52 − 18 | 34 | **退位减**（易错） |
| q-g2m-006 | 小明有29颗糖，又买了14颗，现在几颗 | 43 | 应用题/审题 |

> **故意安排 1–2 道做错**（如进位/退位），才能验「证据优先」：看 Jarvis 说的薄弱点是不是真来自这道错题的 gap，而不是套话。
> 看某题完整题面：`$PY -m agent_platform.learning.cli_student question show q-g2m-002`
