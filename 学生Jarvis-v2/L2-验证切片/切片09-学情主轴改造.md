# 切片09 · 学情主轴改造（错因码轴 → 知识点轴）

> 一个切片 = 一条可证伪假设 + 通过标准 + 最小验证动作。**本文件不得重定义 L0 能力契约。**
> **依据**：[功能架构](../L1-场景域/功能架构-模块与数据流.md) §3/§12（S-A，地基切片）· 决策 D1 · 取舍 T1（迁移不废弃）。

---

## 1. 假设（可证伪）

> **H（技术性）：** 把学情（gap）的主键从「错因码」改为「知识点」后——
> (a) **无错因码**的真实错题（如「方向与位置」某知识点）也能进入学情、形成可追溯的薄弱；
> (b) 已有**加减法**（带错因码）的学情/掌握判定/推题/报告**不退化**，错因码作为知识点下的明细保留；
> (c) 旧数据**自动迁移**（gap_map 是从 attempts 派生重建的，重跑即迁移，不丢证据）。

## 2. 为什么先验这条（动机）

- 它落地 **D1（学情主轴=知识点）**，是后续 S-B（拍照归类入学情）、S-C（家长订正）的**地基**——没有它，任何无错因码的真实错题都进不了学情。
- 验证 L0 的 **I4「场景轨画像」** 在"以知识点为坐标"下依然成立。
- 若证伪（改造导致加减法闭环退化且无法兼容）→ 说明主轴切换代价过高，需回到"错因码为轴 + 仅扩错因表"的老路（推翻 D1）。

## 3. 通过标准（可观测、二元判定 · headless 烟测）

| # | 标准 | 测量方式 | 通过线 |
|---|------|----------|--------|
| 1 | **无错因码错题入学情** | 提交一道 `error_code=None` 的错题（kp=某「方向与位置」KP）→ 查 gap_map | 该 KP 出现一条薄弱，带 `evidence_attempt_ids` |
| 2 | **加减法不退化** | 提交退位错题（带 `BORROW_ERROR`）→ 查 gap_map | 生成对应 KP 薄弱，且 `error_breakdown` 含 `BORROW_ERROR` |
| 3 | **掌握判定不退化** | 同一 KP 连对 3 次 | 该 KP 转 `mastered` |
| 4 | **迁移不丢证据** | 用既有 `g2-stu-01`（旧错因轴数据）重建 | 重建无报错；旧错题证据仍在、按 KP 归并 |
| 5 | **下游不报错** | 推题 `push_engine` + 家长报告 `parent_report` 按新 gap 运行 | 均不抛错，能引用 KP 薄弱 |

## 4. 停止条件（R6）

- **成功即停**：命中全部 5 条 → 沉淀进 L1（功能架构 §12 标 S-A 完成），定为基线，不再优化本切片，转 S-B。
- **证伪即转**：某条不过 → 先尝试兼容修复；若主轴切换确实破坏加减法闭环且不可兼容 → 记录、回退、复盘 D1。

## 5. 最小验证动作（R4/R5）

- [x] 现有能力可直接用：`AttemptService`/`GapMapUpdater`/`GapMapService`/`PushEngine`/`ParentReportService`（rebuild 派生，迁移免脚本）。
- [ ] 需人工补位：无（纯 headless 烟测，不需真人）。
- [ ] 必须新写的最小改动：
  - `contracts.GapEntry`：`error_code` 改可选 + 新增 `error_breakdown: dict[str,int]`。
  - `taxonomy`：新增 `gap_id_for_kp(kp_id)`。
  - `gap_map.rebuild`：改为**按 `knowledge_point_id` 归并**；错题只要有 KP 即入（error_code 可选）；聚合 error_code 入 `error_breakdown`；title 取 KP 目录。
  - 一个 headless 烟测脚本覆盖标准 1–5。

## 6. 真实用户与周期

- 用户：无（地基改造，纯技术烟测；真人价值在 S-B/S-C 验）。
- 周期：1 次烟测通过即可。
- 记录方式：烟测输出 + §7 回填。
