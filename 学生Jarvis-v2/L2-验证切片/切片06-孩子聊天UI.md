# 切片06 · 孩子聊天 UI（build 切片）

> **类型**：build 切片（先建最小可用界面，再验证）。落地 L0 能力契约里的 **C-IO 多模态 I/O** 中"友好的文字交互入口"，替代命令行 `hermes chat`。
> **本文件不得重定义 L0 能力契约。** 仅新增一个面向孩子的 Web 前端，**复用既有 agent 与插件**，不改对话/记忆/学情逻辑。

---

## 0. 要解决的问题

> 现状孩子只能用命令行 `hermes chat` 与小贾对话，对低龄用户极不友好（要开终端、敲命令、看不到气泡）。家长面板 `student_panel.py` 是家长向（周报/KP 审核），**没有孩子可用的聊天界面**。这是产品"能不能真用起来"的门槛。

## 1. 假设（可证伪）

> **H6：** 用一个**复用 `HermesBridge` 的最小 Web 聊天页**（气泡 + 大字 + 发送），就能让孩子以自然、友好的方式与小贾多轮对话，**且不丢失任何既有能力**（跨会话记忆、适龄语气、学情注入、证据优先）。
>
> 若 Web 入口导致**插件/学情上下文丢失**（如 `pre_llm_student_context_hook` 不生效、记忆断裂），或多轮**无法续接**（每条都是新会话），则 H6 证伪 → 需在前端会话管理或 bridge 续接上补齐。

## 2. 最小机制（build 范围，越小越好）

> 一个 FastAPI 应用 + 一个静态 HTML。**不碰 L0，不接 OCR/语音逻辑（仅留按钮占位），不改 agent。**

- **后端 `agent_platform/api/student_chat.py`**：
  - `GET /` → 返回聊天页 HTML。
  - `POST /api/chat {message, session_id?}` → 调 `HermesBridge.ask(message, session_id=...)` → `{reply, session_id}`。
  - 复用完整 hermes agent：memory / calibration / proactive / agent-student 插件均在；`pre_llm_student_context_hook` 按 `default_student_id=g2-stu-01` 自动注入学情。
- **续接**：bridge 已支持 `--resume <session_id>`；前端存住首轮返回的 `session_id` 并回传，保证多轮连续。
- **前端 `templates/student_chat.html`**：暖色、大字号、左右气泡、自适应输入框、回车发送；📷/🎤 按钮先占位（提示"马上就来"），分别对应后续切片的拍照（接 VLM）与语音（接 voice 管线）。

## 3. 通过标准（可观测、二元判定）

| # | 标准 | 测量方式 | 通过线 |
|---|------|----------|--------|
| 1 | **页面能用** | 浏览器打开 `/`，输入并发送 | 出现孩子气泡 + 小贾回复气泡 |
| 2 | **学情注入不丢** | 问"我哪个知识点没掌握" | 引用既有 gap（退位/审题），与命令行一致 |
| 3 | **多轮续接** | 连续 2-3 轮、含指代（"那再来一题"）| 上下文连贯，非各自独立 |
| 4 | **适龄语气不退化** | 说"不想写作业了" | 先共情、不硬拉回（同切片01 基线）|
| 5 | **孩子友好度** | 让孩子/家长实际点用 | 主观：愿意用、比命令行友好 |

## 4. 停止条件（R6）

- **成功即停**：标准 1-4 过 → 文字 UI 沉淀进 L1 基线；再做 OCR（切片05/拍照接 📷）与语音（接 🎤）。
- **证伪即修**：学情/记忆丢失或多轮断裂 → 优先修 bridge 续接 / 上下文注入，不动 L0。

## 5. 真实用户与周期

- 用户：同一名孩子 `g2-stu-01`（盈熙）+ 家长旁观。
- 周期：起服务后 1 次会话验证（覆盖标准 1-5）。
- 记录方式：①能否正常收发 ②学情是否被引用 ③多轮是否连贯 ④语气 ⑤孩子/家长主观友好度。

## 6. 运行方式

```bash
PY=/home/administrator/.hermes/hermes-agent/venv/bin/python
$PY -m uvicorn agent_platform.api.student_chat:app --host 127.0.0.1 --port 8771
# 浏览器打开 http://127.0.0.1:8771/
```

## 7. 结果（2026-06-22 回填 · 真人浏览器验证通过）

- 结论：**✅ 通过（H6 成立）** —— 复用 `HermesBridge` 的最小 Web 聊天页，孩子能自然友好地多轮对话，且**不丢任何既有能力**。
- 证据（真人浏览器 `http://127.0.0.1:8771/` 实测对话）：
  - **标准1 能用 ✅**：气泡收发正常，回复顺畅。
  - **标准2 学情注入不丢 ✅（超预期）**：问「数学哪些知识点掌握不好」→ 准确引用真实漏洞 **退位错误 + 审题列式错误**；并认出身份（刘盈熙/8岁/二年级）与喜好（画画/跳舞）——**学情（pre_llm hook）+ 跨会话记忆** 经 Web 路径双双注入。
  - **标准3 多轮续接 ✅**：跨多轮连贯，记得前文身份/偏好。
  - **标准4 语气不退化 ✅**：说「我现在不想学习写作业」→「哈哈，好啦好啦，不想写作业就不写～那你想干什么呀？」**先共情、给选项、不硬拉回**（同切片01 基线）。
  - **标准5 友好度 ✅（经验性）**：全程温暖、个性化、有 emoji，明显比命令行友好。
- **关键修复（bridge 收尾不退出）**：`hermes chat` 在非交互管道子进程下会**完成本轮并打印结果、但进程不退出**（命令行经 PTY 能正常退出）。原 `HermesBridge` 用 `subprocess.run` 阻塞等退出 → 必然超时 504。改为：输出重定向到临时文件 + 独立进程组 `Popen`，**轮询到 STDOUT 产出且稳定即判定完成、回收进程组**，并从 STDERR 解析 `session_id`。e2e 实测 ~21s 返回。
- 运行方式见 §6（`uvicorn ... --port 8771`，浏览器开 `http://127.0.0.1:8771/`）。
- 沉淀：**Web 文字聊天入口 + bridge 鲁棒运行** 确立为基线，记入 L1。
- 下一步：接 📷（OCR/切片05，需视觉 key）与 🎤（语音，接 voice 管线）。
