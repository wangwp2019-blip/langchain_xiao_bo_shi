# 切片08 · 已批改作业入学情（验证 + build 切片）

> **类型**：先验证（VLM 能否读懂批改），通过后 build（识别→确认→入学情）。延续 [切片05](切片05-拍照识题.md) 的图像输入，把 prompt 从「抄题」升级为「理解」。
> **本文件不得重定义 L0 能力契约。** 图像理解只产出结构化结果，入学情仍复用 [切片04](切片04-真实题纳入学情.md) 的 `attempt_submit_freeform`；保持证据优先与可追溯。

---

## 0. 背景（来自用户反馈 2026-06-22）

- 切片05 的 `/api/ocr` prompt 写死为「只抄题目原文」，所以上传**老师已批改的作业**时，孩子的答案与对错被忽略——不是模型不行，是 prompt 限定了目的。
- 上传图片**意图多样**：A 纯题目 / **B 已批改作业** / C 已作答未批改 / D 课本知识页 / E 无关图。本切片先攻**最高价值的 B**。
- B 的价值：老师已判对错，VLM 读出「哪题错、孩子写了啥」即可**不用孩子重做**直接更新学情，复用切片04 freeform。

## 1. 假设（可证伪）

> **H8：** VLM（Qwen-VL）能从「已批改作业图」**准确读出结构化信息**（题目 / 孩子答案 / 对错），经「识别 → 人确认 → 入学情」就能把错题纳入 gap_map（复用切片04），**且不污染证据**（人确认 + `source=freeform` 审计）。
>
> 若 VLM **读对错/答案不稳**（对勾看成叉、看错数字、漏题），即便有人确认也成本过高/体验差 → H8 证伪 → 降级（仅做题目转写 + 孩子口述对错 / 家长手填）。

## 2. 方案（理解 + 结构化 + 自动判类型可纠正）

- **理解型 prompt**：让 VLM 输出结构化 JSON：
  ```json
  {
    "image_type": "graded_homework | blank_problem | answered_unmarked | knowledge_page | other",
    "items": [
      {"stem": "题目原文", "student_answer": "孩子写的", "is_correct": true, "teacher_mark": "✓/✗/无"}
    ]
  }
  ```
- **路由**：模型自动判 `image_type` → 前端展示「我看到这是【已批改作业】，识别到 N 题，其中 X 题错了」→ 用户**可一键纠正类型/逐题改对错**。
- **入学情**：用户确认后，对每道 `is_correct=false` 的题调 `attempt_submit_freeform`（题面=stem、答案=student_answer、correct=false、错因由 LLM/规则给并白名单校验），写入 gap_map；`source=freeform` 标记「据批改图」。
- **证据优先**：识别结果一律「先展示、人确认、再入学情」，不盲信 VLM。

## 3. 通过标准（可观测、二元判定）

| # | 标准 | 测量方式 | 通过线 |
|---|------|----------|--------|
| 1 | **判对类型** | 传已批改图 | `image_type=graded_homework` |
| 2 | **读准结构** | 人工比对 题/孩子答案/对错 | 题面与对错 100% 对、答案数字不错（小样本）|
| 3 | **入得了学情** | 确认后看 gap/attempt | 错题经 freeform 进 gap_map、`source=freeform` |
| 4 | **不污染证据** | 检查是否盲信 | 必经人确认；gap 可溯源、不夸大为实测 |
| 5 | **家长愿用** | 主观 | 比手填省事、信得过 |

## 4. 停止条件（R6）

- **成功即停**：标准1-4 过 → 已批改作业入学情沉淀进 L1；再逐步扩 C/D 等意图。
- **证伪即降级**：读对错不稳 → 仅转写题目 + 口述/手填对错。

## 5. 真实用户与周期

- 用户：孩子 `g2-stu-01`（盈熙）+ 家长。
- 周期：拿到 1-2 张真实已批改作业图后，先 headless 验准确率（标准1-2），再接确认+入学情（标准3-5）。
- 记录：①类型判对率 ②题/答案/对错读准率 ③入学情是否通 ④是否盲信 ⑤家长愿用。

## 7. 结果（2026-06-23 回填 · 核心成立 + 两个关键发现）

- 结论：**🟡 核心假设成立**——VLM（qwen3-vl-plus，16.6s）对真实语文已批改作业：**判对类型 `graded_homework` ✅**，并从「背一背填一填」逐题读出**孩子答案 + 对错**（如「映日荷花别样红 ✓」「一行白鹭上青天 ✗」）。证明能读懂批改、产出结构化信息。
- 但暴露两个关键问题：

### 发现 F6：密集网格弱 + 对错可信度需人核
- 「看拼音写词语」大网格只读了**部分**、未逐格判对错——密集手写网格是弱项。
- 个别对错需复核（如疑似正确的诗句被判 ✗，可能老师划错或 VLM 误读）。→ **印证「识别→人确认→再入学情」必须保留，不能盲信。**

### 发现 F7（架构性）：语文错题套不进现有数学学情管线
- 现有 gap_map / 错因表（`CARRY_ERROR/BORROW_ERROR/READING_ERROR`…）是**为二年级数学算术建的**；语文错（背诵/拼音/字词）无法映射到该错因白名单。
- 即识图能读出语文错题，但 `attempt_submit_freeform` 的错因校验接不住 → **「入学情」这一步对语文当前不通**。

### 补充验证（2026-06-23 · 真实数学批改图：二年级下「方向与位置」）

- 识图：类型判对 `graded_homework`✅；7 道方向填空的**题目+孩子答案读准**✅；对错几乎全判"对"（该卷约 97 分，基本相符，但红笔具体订正处人/机都难从照片确认）→ 再次印证 **F6 对错须人确认**。
- **F7 被进一步放大（关键）**：错因表 `student_learning.yaml` 现仅覆盖**二年级加减法**（`CARRY/BORROW/NO_CARRY/READING_ERROR`）。**「方向与位置」无对应错因码**——即便对错读准，freeform 错因白名单也接不住，进不了 gap_map。
- 结论：**能端到端入学情的目前只有「加减法计算」批改图**；用户手上两张真实作业（语文、方向与位置）均在现有学情模型之外。**瓶颈不是识图（已验证可用），是学情模型只覆盖二年级加减法。**

### 取舍与下一步
- **数学已批改作业**：识图 + freeform 学情管线**今天就能端到端跑通**（错因表匹配）。
- **语文已批改作业**：需要先建**语文错因维度/学情模型**（F7），否则只能"识别+展示"，进不了 gap_map。
- 建议：① 先用**一张数学已批改作业**验证完整闭环（识别→确认→入 gap_map）；② 语文作为独立方向，先做"识别+家长查看"，学情建模另立切片。
