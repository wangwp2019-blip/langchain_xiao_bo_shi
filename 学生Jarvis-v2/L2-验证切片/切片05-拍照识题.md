# 切片05 · 拍照识题（验证 + build 切片）

> **类型**：先验证（识别准不准、值不值得），通过后再 build（接 📷 到聊天入口）。落地 L0 能力契约 **C-IO 多模态 I/O** 的「图像输入」。
> **本文件不得重定义 L0 能力契约。** 图像仅用于**转写成题目文字**，下游辅导仍走既有文字管线（学情/漏洞/AnswerGate 不变）。

---

## 0. 背景与关键事实

- 目标：把「家长打字输入真题」自动化——孩子/家长**拍张作业照片**，Jarvis 自动抄出题目，进入答疑/做题闭环。
- **关键事实（2026-06-22 实测）**：**DeepSeek 官方 API 不支持图片输入**（`deepseek-v4-pro/flash/chat` 均返回 400 `unknown variant image_url`）。其"识图"是网页/App 端能力，不在 API。故需独立视觉模型。
- 选型：**用 VLM（大模型识图）而非传统 OCR**——孩子作业含手写、应用题、竖式，OCR 对手写/数学排版弱；VLM 能"看懂"并忠实抄题。
- 复用：`perception/vlm.py` 的 OpenAI 兼容适配器（`describe(image, question)`），接 **DashScope Qwen-VL**（`DASHSCOPE_API_KEY`，base_url `https://dashscope.aliyuncs.com/compatible-mode/v1`）。

## 1. 假设（可证伪）

> **H5：** 用 VLM 以「抄出题目原文」模式识别孩子的真实作业照片，**转写准确率足够高**（题面、数字、运算符不出错），就能替代家长打字、把真题送进既有答疑/做题闭环，**值得做**。
>
> 若**数字/运算符/题意频繁出错**（如 30 抄成 80、加减看反），导致后续辅导基于错题面 → H5 证伪 → 降级（家长复核转写 / 暂缓拍照）。

## 2. 架构（图像只转文字，不绕过学情）

```
拍照 → /api/ocr → VLM.describe(img, "抄出题目原文") → 题面文字
     → 填入聊天输入框（家长/孩子确认可改）→ 发给 hermes agent（带学情）→ 答疑/做题
```

- VLM 只做**转写**；辅导、判分、漏洞仍由 deepseek + agent-student 完成，学情链路零改动。
- hermes 的 `--image` 用不上（deepseek 看不了图）。

## 3. 通过标准（可观测、二元判定）

| # | 标准 | 测量方式 | 通过线 |
|---|------|----------|--------|
| 1 | **接得通** | 配好 key，headless 跑 `smoke_ocr_homework` | VLM 返回非空转写、无鉴权/格式错 |
| 2 | **转写准** | 3-5 张真实作业照片，人工比对 | 题面/**数字/运算符**逐字正确率 ≥ ~90%，关键数字不错 |
| 3 | **进得了闭环** | 把转写文字当题面走 freeform/答疑 | 与切片03/04 一致地答疑、可纳入学情 |
| 4 | **值得用** | 家长主观 | 比自己打字省事、愿意用 |

## 4. 停止条件（R6）

- **成功即停**：标准1-3 过 → 接 📷 到聊天入口（build），拍照识题沉淀进 L1。
- **证伪即降级**：转写不稳 → 家长复核模式（转写后必须确认再发）/ 换更强视觉模型 / 暂缓。

## 5. 真实用户与周期

- 用户：孩子 `g2-stu-01`（盈熙）+ 家长。
- 周期：拿到 key + 几张真实作业照后，先 headless 验准确率（标准1-2），再接 UI 验闭环（标准3-4）。
- 记录：①能否调通 ②逐字/数字准确率 ③闭环是否顺 ④家长是否愿用。

## 6. 运行方式（headless 验证）

```bash
# 1) 在 ~/.hermes/.env 加：DASHSCOPE_API_KEY=sk-xxx
# 2) 准备照片，例如 /mnt/c/Users/Administrator/Desktop/作业1.jpg
PY=/home/administrator/.hermes/hermes-agent/venv/bin/python
$PY -m agent_platform.perception.smoke_ocr_homework /mnt/c/.../作业1.jpg
# 输出：VLM 抄出的题目原文，人工比对准确率
```

## 7. 结果（2026-06-22 回填 · 接口通 + 首样通过）

- 结论：**🟢 接口验证通过，首张样本转写准确**（标准1、2 在该样本成立）；建议再补手写算式样本扩大信心。
- 证据（DashScope `qwen3-vl-plus`，2.1s）：
  - **标准1 接得通 ✅**：`DASHSCOPE_API_KEY` 有效，`provider=openai_compatible model=qwen3-vl-plus`，返回非空、无鉴权/格式错。
  - **标准2 转写准 ✅（印刷体样本）**：原题「3. 用36根小棒可以摆（ ）个六边形，也可以摆（ ）个菱形。」→ 抄写「3. 用 36 根小棒可以摆（ ）个 ⬢，也可以摆（ ）个 ◇。」**题号、数字 36、括号、文字逐字正确，且识别出六边形/菱形几何图形**。
- 样本量：1 张（印刷体）。**待补**：手写体 + 含 +−×÷ 的算式题，确认数字/运算符在手写场景仍稳。
