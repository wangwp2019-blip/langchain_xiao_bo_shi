# 切片02 · 学习闭环自然融入

> 一个切片 = 一条可证伪假设 + 一个真实用户的通过标准 + 最小验证动作。
> **本文件不得重定义 L0 能力契约。** 本切片建立在 [切片01](切片01-二年级自然对话.md) 已验证的对话基线之上。

---

## 1. 假设（可证伪）

> **H2：** 在切片01 已验证的「对话底座」之上引入真实做题后，二年级孩子做 3–5 道题、用 2–3 天，Jarvis 能做到：
> （a）**证据优先**——它说出的任何「你这里薄弱 / 我们练这个」都能追溯到孩子真实的 attempt/gap，不臆测；
> （b）**自然融合**——做题前后仍是对话流（会引用对话记忆、兴趣、心情），不是冷冰冰报对错；
> （c）**不硬推**——孩子做完一题不想继续时，Jarvis 仍共情、不机械塞下一题。
>
> 若做起题来就退化成「报对错 / 机械刷题 / 凭空下结论」的学习机，则 H2 证伪。

## 2. 为什么先验这条（动机）

- 这是整个产品的**命门**：切片01 证明了「对话学伴感」成立；H2 要回答的是——**把学习能力接进来，会不会把好不容易建立的学伴感破坏掉**，退回成生硬学习机。这正是项目最初担忧的核心。
- 它验证 L0 的 **C-KNOW（领域答疑/学情）** 与 **C-DIALOG / C-MEM / C-CALIB** 能否在同一轮里共存：学情结论必须由证据驱动（C-GOV 的「不臆测」），而表达仍走对话与共情。
- 若证伪：要推翻的不是 L0，而是 L1 学习工具的**接入方式**（如 AnswerGate 规则、做题节奏、证据呈现），属场景层调整，**不动内核**。


## 3. 通过标准（可观测、二元判定）

| # | 标准 | 测量方式 | 通过线 |
|---|------|----------|--------|
| 1 | **证据优先** | 抽查 Jarvis 给出的「薄弱点/练习建议」，逐条回溯是否能在 `gap_map_query` / `attempt_submit` 记录里找到来源 | 抽查项 **100% 可溯源**，0 条凭空臆测 |
| 2 | **自然融合** | 看做题前后是否仍引用对话记忆/兴趣/心情，而非纯报对错 | 至少 1 次自然引用历史对话；孩子主观「不像机器/学习机」 |
| 3 | **不硬推** | 孩子做完一题说「不想做了」时的反应 | 先共情、给出口，不机械推下一题（与切片01 标准3 一致不回退） |
| 4 | **学情可见且可信** | 结束后能否产出一条「孩子今天卡在 X，证据是这道题」的可追溯结论给家长/你 | 能产出 ≥1 条带证据的学情结论，且你认可其真实 |

## 4. 停止条件（R6）

- **成功即停**：命中全部通过标准 → 把「做题闭环 + 证据优先 + 对话融合」沉淀进 L1 作为基线，**不再优化本切片**。
- **证伪即转**：未命中 → 改 L1 接入方式（AnswerGate 规则 / 做题节奏 / 证据呈现），**不动 L0**。

## 5. 最小验证动作（R4/R5）

> 复用切片01 的对话底座 + WoZ，只新增「真实做题」这一变量。先用现有工具，不提前写新代码。

- [x] 现有能力可直接用：`hermes chat` + 已启用插件（含 **agent-student**，doctor 已验证 `✓ agent_student`）；`attempt_submit` / `gap_map_query` / `student_context_get` / `push_queue_peek` / `student_answer_gate` 均**作为模型工具注册**；`pre_llm_student_context_hook` 每轮自动注入画像+漏洞+队头题号+AnswerGate。
- [x] **环境已备好**：学生上下文 `g2-stu-01` 已初始化、题库 20 题已导入、推题队列已用单元题填 10 题、`default_student_id` 已设（`hermes chat` 免 env 自动认人）。
- [ ] 验证方式 = **自然语言优先**：孩子在对话里口头作答，**Jarvis 自己调 `attempt_submit` 判分落漏洞**；人不手敲答案。CLI 仅用于**事后核对证据**（`attempt list` / `gap list`）与**兜底补录**（模型漏调工具时）。
- [ ] 人工补位（WoZ）：仅「转述」——大人把孩子的话打进 chat、把 Jarvis 回复念给孩子；以及故意安排 1–2 道错题以验证据优先。
- [ ] 不做：不提前搭推题算法、不做 UI。证据优先与对话融合用现有 AnswerGate + 对话验证即可。

## 6. 真实用户与周期

- 用户：与切片01 同一名真实二年级孩子（自家/家人），延续已建立的对话记忆。
- 周期：2–3 天，3–5 道真实题（语文或数学，任选其一先验）。
- 记录方式：每次做题后记录三件事——①Jarvis 的学情结论 ②该结论的证据来源 ③孩子主观感受（像不像在和学伴聊，而非被机器考）。

## 7. 结果（2026-06-21 回填）

- 结论：**✅ 通过**——H2 四条标准全部命中（真实二年级孩子「盈熙」单次会话验证）。
- 证据：
  - **标准1 证据优先 ✅**：问「哪个知识点没掌握」时先调 `gap_map_query`，准确区分已掌握（不进位/进位加法）vs 未掌握（退位减法），指名引用错题 52−18、错因「借位后忘减1」、gap 编号 `gap-borrow-error`，**零臆测**。
  - **标准2 自然融合 ✅**：温暖学伴语气 + 叫出名字 + 按「太简单了」自适应加难度；`attempt_submit` 全程模型自动调用，3 次作答题号/答案/判定全部正确落库。
  - **标准3 不硬推 ✅**：说「不想做了，有点累了」→ 先认同「今天你已经很棒了」+ 给小结 + 建议放松，零强推下一题。
  - **标准4 学情可信 ✅**（家长判定）：周报「3 次 67%、薄弱=退位、证据可溯源」核心属实。
- 发现（转 L1 改进，**不动 L0**）：
  - **F1**：缺「按题号取题面」工具（`push_queue_peek` 只给题号）→ 出题时模型 grep/find/读 seed 源文件，慢且脆。
  - **F2**：维度模型漏映射 `BORROW_ERROR`/`CARRY_ERROR` → 家长报告「能力维度分」全 0、失真。
- 下一步：沉淀「做题闭环 + 证据优先 + 对话融合」为 L1 基线；F1/F2 列入 L1 改进项。
