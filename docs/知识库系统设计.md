### 1.3 数据流

```

  → MinerU 解析：PDF → Markdown（保留表格/公式/排版）
  → 提取纯文本 → normalize_text
  → LlamaIndex 切分（chunk_size=500, overlap=50）
  → BGE-M3 Embedding（1024 维）
  → Milvus insert（metadata: grade=3, subject=数学, doc_id, ...）

用户提问："三年级数学——分数怎么比较大小？"
  → POST /api/chat/stream（JWT 鉴权）
  → Agent 调用 search_knowledge_base 工具
  → LlamaIndex retriever 检索（按 grade=3, subject=数学 过滤）
  → top_k=4 片段注入 prompt 上下文
  → LLM 生成回答（引用知识库来源）
  → SSE 流式返回
```

## 三、MinerU 集成设计

### 3.1 MinerU 概述

[MinerU](https://github.com/opendatalab/MinerU) 是 OpenDataLab 开源的高精度文档解析引擎：
- **输入**：PDF / 图片(PNG,JPG,WEBP) / DOC / DOCX / PPT / PPTX
- **输出**：Markdown + JSON（保留表格 HTML、公式 LaTeX、阅读顺序）
- **能力**：OCR、表格识别、公式识别、多栏排版、跨页表格合并、页眉页脚去除

两种 API 模式：

| 维度 | 精准解析 API | 轻量解析 API |
|------|------------|------------|
| Token | 需要（官网申请） | 不需要 |
| 文件大小 | ≤ 200MB | ≤ 10MB |
| 页数 | ≤ 600 页 | ≤ 20 页 |
| 批量 | 支持（≤ 200 文件） | 单文件 |
| 输出 | Markdown + JSON + docx/html/latex | Markdown |
| 模型 | pipeline / vlm（推荐） | 轻量 pipeline |

### 3.2 集成方案

```python
# app/knowledge_library/mineru_parser.py（新增）

"""
MinerU 文档解析适配器。

双模式：
1. 在线 API（MINERU_API_TOKEN 配置后启用）：
   - 适合生产环境，无需本地 GPU
   - 精准模式：POST /api/v4/extract/task → 轮询结果
   - 轻量模式：POST /api/v1/agent/parse/file → 轮询结果

2. 本地 SDK（pip install mineru）：
   - 适合开发/离线环境
   - do_parse() 同步调用，本地 OCR + 版面分析
   - 需要 PyTorch + GPU（CPU 也可但慢）

降级链：MinerU API → MinerU 本地 SDK → pypdf/python-docx（现有）
"""
```

### 3.3 解析流程

```
上传文件
  │
  ├─ 文件 ≤ 10MB 且无 Token？─→ MinerU 轻量 API（免登录）
  │
  ├─ 有 MINERU_API_TOKEN？
  │    ├─ 文件 ≤ 200MB 且 ≤ 600 页？─→ MinerU 精准 API（vlm 模式）
  │    └─ 超限？─→ 分片处理或提示用户
  │
  ├─ 已安装 mineru SDK？
  │    └─ 本地 do_parse()（pipeline 模式）
  │
  └─ 降级到现有 parser.py
       ├─ PDF → pypdf
       ├─ 图片 → LLM OCR
       └─ Word → python-docx
```

### 3.4 API 调用示例（在线精准模式）

```python
import requests

MINERU_BASE = "https://mineru.net"
MINERU_TOKEN = settings.mineru_api_token  # 从 .env 读取

def mineru_extract_by_url(file_url: str, model_version: str = "vlm") -> str:
    """提交 URL 到 MinerU 精准解析，轮询取回 Markdown。"""
    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {MINERU_TOKEN}",
    }
    # 1. 创建任务
    resp = requests.post(
        f"{MINERU_BASE}/api/v4/extract/task",
        headers=headers,
        json={"url": file_url, "model_version": model_version, "enable_formula": True, "enable_table": True},
    )
    task_id = resp.json()["data"]["task_id"]

    # 2. 轮询（间隔 3s，超时 300s）
    for _ in range(100):
        resp = requests.get(
            f"{MINERU_BASE}/api/v4/extract/task/{task_id}",
            headers=headers,
        )
        data = resp.json()["data"]
        if data["state"] == "done":
            # 3. 下载 Markdown 结果
            md_url = data["full_zip_header"]  # ZIP 包含 .md + .json
            return _download_and_extract_markdown(md_url)
        elif data["state"] == "failed":
            raise RuntimeError(f"MinerU 解析失败: {data.get('err_msg')}")
        import time; time.sleep(3)
    raise TimeoutError("MinerU 解析超时（5 分钟）")
```

## 四、RAG 知识库增强

### 4.1 文档切分策略

```python
# 现有：RecursiveCharacterTextSplitter（按字符数切分）
chunk_size = 500      # 每块 500 字符
chunk_overlap = 50    # 重叠 50 字符

# 增强：MinerU 输出 Markdown，按结构切分
# 1. MinerU 输出的 Markdown 天然有标题层级（# / ## / ###）
# 2. 先按标题切分，再对过长段落做字符切分
# 3. 保留表格和公式完整性（不切断 <table> / $$ 公式 $$）
```

切分流程：

```
MinerU Markdown 输出
  │
  ├─ 按标题层级切分（# → ## → ###）
  │    每个标题段落作为一个 chunk
  │
  ├─ 表格/公式保护：不切断 <table>...</table> 和 $$...$$
  │
  ├─ 过长段落（> 1000 字符）：RecursiveCharacterTextSplitter 二次切分
  │
  └─ 短段落（< 20 字符）：合并到上一块
```

### 4.2 混合检索（向量 + 关键词）

```
用户查询 "分数比较大小"
  │
  ├─ 向量检索（BGE-M3 Embedding）
  │    Milvus top_k=12（cosine similarity）
  │
  ├─ 关键词检索（BM25 / 中文 bigram）
  │    Milvus全文检索 或 langchain_community BM25Retriever
  │    top_k=12
  │
  ├─ 融合排序（Reciprocal Rank Fusion）
  │    score = 1/(60+rank_vec) + 1/(60+rank_bm25)
  │
  ├─ 元数据过滤（前置）
  │    grade=3 AND subject=数学
  │
  └─ 取 top_k=4 注入 LLM prompt
```

### 4.3 元数据 schema

每个 chunk 的 metadata：

```python
{
    "doc_id": "kb-a1b2c3d4e5f6",      # 文档唯一 ID
    "title": "三年级数学-分数初步认识",  # 文档标题
    "source": "三年级数学-分数初步认识", # 显示名
    "chunk_id": 3,                     # 切片序号
    "grade": 3,                        # 年级 1-6
    "subject": "数学",                  # 学科
    "upload_type": "pdf",              # 上传类型
    "filename": "分数初步认识.pdf",     # 原始文件名
    "heading": "二、分数比较大小",       # MinerU 解析的标题（新增）
    "page": 5,                         # 来源页码（MinerU 提供，新增）
}
```

### 4.4 LLM 对话引用来源

```python
# tools.py: search_knowledge_base 工具增强

@tool
def search_knowledge_base(query: str, grade: int | None = None, subject: str | None = None) -> str:
    """搜索知识库，返回相关片段及其来源。"""
    hits = retrieve(query, top_k=4, grade=grade, subject=subject)
    if not hits:
        return "知识库中暂无相关内容。"

    parts = []
    for i, h in enumerate(hits, 1):
        source = h.get("source", "")
        score = h.get("score", 0)
        parts.append(f"【片段{i}】（来源：{source}，相似度：{score:.0%}）\n{h['text']}")
    return "\n\n---\n\n".join(parts)
```

Agent system prompt 增加：

```
回答时优先使用知识库检索结果。如果引用了知识库内容，在回答末尾标注来源：
📚 来源：[文档名] · 相似度 85%
```

## 十、安全与合规

| 维度 | 措施 |
|------|------|
| **鉴权** | 所有 API 端点要求 JWT（12h 有效 + 自动续期） |
| **文件大小** | 单文件 ≤ 200MB（MinerU 限制），本地限制可配 |
| **文件类型** | 白名单：txt/md/pdf/png/jpg/jpeg/webp/doc/docx/ppt/pptx |
| **内容安全** | 上传文本经 `safety.py` 敏感词过滤后再入库 |
| **数据隔离** | JWT principal → user_id，知识库按用户隔离（多租户预留） |
| **速率限制** | 上传 API 限流 10 次/分钟，检索 API 限流 60 次/分钟 |
| **审计日志** | 上传/删除/重建操作记录到 audit.log |
