## 1. 验证目标与通过标准

### 1.1 要证明什么

| 层级 | 证明内容 |
|------|----------|
| **L2 Hermes** | 6 个 text 插件加载；`agent_*` 工具可调用；进化 hooks 注册 |
| **L3 agent_platform** | 记忆/Wiki/工具治理/主动/校准/行为/进化 **产品语义** 按设计工作 |
| **数据主权** | 记忆可落盘、面板可删、tombstone 后 search 不命中 |
| **跨会话** | 退出 `hermes chat` 后仍能通过 `agent_memory_search` 召回（mock 落盘或 MemVerse） |

### 1.2 总通过标准

| 阶段 | 必过？ | 通过判据 |
|------|--------|----------|
| Phase 0 | ✅ | `check_wsl_env` 无致命项；`hermes doctor` 中 6 个 `agent_*` 工具集为 ✓ |
| Phase 1 | ✅ | `run_wsl_text_full_acceptance.sh` 末尾 `All text-mode automated checks passed.` |
| Phase 2 | ✅ | `verify_m2_memory_persist.sh` → `verify_m2_memory_persist: PASS` |
| Phase 3 | 推荐 | 三面板各完成 1 条增删改 |
| Phase 4 | ✅ | 下表 **P4-M2 ~ P4-C7** 全部 ☐→✅ |
| Phase 5 | 推荐 | P5-1 combined_recall；P5-2 进化钩子可见 |
| Phase 6 | 可选 | `--memverse` 自动化 + 1 轮 Hermes 写入 |

### 1.3 不在本方案内

- US-2 视觉 / Reachy（需 OpenCV、硬件）
- M1 麦克风全链路 / TTS 时延 bench
- 7 天自用日记（产品运营，非功能回归）

### 1.4 关键约定（必读）

1. **产品长期记忆** 只用 `agent_memory_write` / `search` / `delete`，**禁止**用 Hermes 内置 `memory` 做 US-3 验收。  
2. **mock 落盘** 默认：`memory.yaml` → `mock.persist_path: skills_data/memory_store.json`。  
3. 每个 Hermes 用例后，用 **CLI 二次确认**（见各节「CLI 验证」），不只看模型自然语言。

### 6.1 M2 长期记忆（P4-M2）

#### P4-M2-1 写入

**你说**（复制）：

```text
请只使用 agent_memory_write（禁止使用内置 memory 工具），
写入：用户偏好：回复风格简短直接。
category 使用 preference。
完成后告诉我 record_id。
```

| 检查 | 预期 |
|------|------|
| 工具名 | `agent_memory_write` |
| 返回 | 含 `record_id` / success |

**CLI 验证**（Hermes **不要关**）：

```bash
PYTHONPATH=. python agent_platform/memory/cli_memory.py search "简短"
```

期望：`hits` ≥ 1。

#### P4-M2-2 跨会话召回

1. 退出 `hermes chat`（Ctrl+C 或 exit）  
2. 新开：`hermes chat`  
3. **你说**：

```text
我的回复风格偏好是什么？请先用 agent_memory_search 查长期记忆，再简短回答。
```

| 检查 | 预期 |
|------|------|
| 工具名 | `agent_memory_search` |
| 回答 | 提到「简短」类偏好 |
| 失败若空 | 先 CLI：`cli_memory.py search "简短"` 应有 hits；若无 active 记录需重做 P4-M2-1；勿与 Phase 3 面板删除（tombstone）混淆 |

#### P4-M2-3 删除与 tombstone

1. `agent_memory_search` 找到 record_id，或从面板复制  
2. **你说**：「请用 agent_memory_delete 删除刚才那条回复风格偏好，record_id=…」  
3. **新开** `hermes chat`（清掉本会话里「刚才说过简短」的历史）  
4. **你说**（复制）：

```text
请只用 agent_memory_search 查「回复风格」。
若 count=0 或 hits 为空，必须明确说长期记忆里没有该偏好，不要猜。
```

5. CLI 验收：`search "简短"` → `hits: []`

| 检查 | 预期 |
|------|------|
| CLI | `hits: []`（你当前已满足 ✅） |
| 工具 | `agent_memory_delete` 成功；`memory_store.json` 对应条 `status: tombstoned` |
| Hermes 回答 | 应说「长期记忆未找到」类表述 |

**为何删了 Hermes 仍答「简短直接」？（常见误判）**

| 来源 | 说明 |
|------|------|
| M2 长期记忆 | 删除后 search 为空 — **这才是 P4-M2-3 的判据** |
| M7 行为设定 | `behavior.yaml` 默认规则含「回复尽量简短直接」— 是**助手行为档**，不是用户偏好记忆 |
| 同会话历史 | 删除前聊过偏好，模型可能「回忆」对话，需 **新开 hermes chat** |
| 内置 memory | 若出现工具名 `memory` 而非 `agent_memory_search`，属 L2 误用，不算 M2 通过 |

**磁盘核对**：

```bash
grep -A2 '回复风格简短直接' skills_data/memory_store.json | grep status
# 应为 "tombstoned"
```

## 附录 A — 故障排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `set: invalid option name` | `.sh` CRLF | `bash scripts/fix_sh_crlf.sh` |
| `No module named agent_platform` | 未 source bashrc | `export PYTHONPATH=$AGENT_COMMUNITY_ROOT` |
| `hermes chat` 无 `agent_*` 工具 | 插件未启用 | `install_hermes_plugins_wsl.sh` + `enable_hermes_text_plugins.sh` |
| 写了偏好但 search 空 | 用了内置 `memory` | 强制 `agent_memory_write`；见 P4-M2 |
| 跨会话 search 空 | 无 persist_path | 检查 `memory.yaml` mock.persist_path |
| 面板空但 CLI 有数据 | 面板未共用 persist（旧版） | 确认 `memory_panel.py` 保留 mock.persist_path |
| 首次 chat 极慢 | lazy 依赖 | 等待 1–2 分钟 |
| MCP L2 无草稿 | mock transport | 正常；smoke 可测；stdio 见 `mcp.yaml` |
| curriculum 不注入 | 有 skill 召回 | 正常；无关 query 才注入；或 `inject_always: true` |
| doctor 大量 ⚠ | 可选工具缺 key | 忽略；见 [模型与产品层边界.md](./模型与产品层边界.md) |
