# 分阶段测试说明

本项目提供 **S0–S8** 九个阶段的渐进式测试，从离线基础设施到端到端 Agent，便于定位问题发生在哪一层。

## 阶段一览

| 阶段 | 名称 | 是否需 API | 验证内容 |
|------|------|------------|----------|
| **S0** | 配置与导入 | 否 | 版本、路径、`.env`、核心模块 import |
| **S1** | RAG 基础设施 | 否 | 文档切分、本地向量库、关键词检索、分词 |
| **S2** | 记忆系统 | 否 | FileBackedStore 落盘恢复、checkpointer、store、连接池关闭 |
| **S3** | 工具层 | 否 | calculator、日期、工具列表、search_knowledge_base（keyword） |
| **S4** | Agent 组装 | 否 | **旧版**：`create_agent` 编译；**LangGraph 版**：StateGraph 拓扑 |
| **S5** | RAG 在线链路 | 否 | keyword 建索引、retrieve、format_context（+ 图节点） |
| **S6** | 循环引擎 | 否 | Worker 重试、Scheduler 触发、限流/退避/超时 |
| **S7** | HTTP API | 否 | `/api/health`、`/`、`/api/ingest`（TestClient） |
| **S8** | 端到端集成 | **是** | 完整 Agent：记忆、计算、流式、学习卡片 |

## 快速运行

```bash
# 默认：S0–S7（不消耗 LLM，推荐 CI / 本地先跑）
python run_all_tests.py

# 含 S8 端到端（需有效 LLM Key；Embedding 建议 keyword 后端）
python run_all_tests.py --api

# 只跑某一阶段
python run_all_tests.py --stage S4

# 跑一段区间
python run_all_tests.py --from S2 --to S6
```

## 推荐执行顺序（新手排查）

```
S0 配置 → S1 切分/向量 → S2 记忆 → S3 工具
    → S4 图/Agent → S5 建索引检索 → S6 引擎 → S7 API → S8 端到端
```

某阶段失败时，**先修当前阶段再往后跑**，避免上层测试被下层问题误导。

## 与旧脚本的关系

| 新阶段 | 等价/覆盖的旧脚本 |
|--------|-------------------|
| S0–S3 | `local_selftest.py` 大部分 |
| S4 | `graph_selftest.py`（LangGraph 版） |
| S6 | `engines_selftest.py` |
| S8 | `smoke_test.py` |

`run_all_tests.py` 是**统一入口**；旧脚本仍保留，可单独运行。

## S8 前置条件

1. 教程根目录或项目 `.env` 中配置有效 **LLM Key**
2. 若 `KIDS_VECTOR_BACKEND=local`，还需有效 **Embedding Key**；否则在 `.env` 设：
   ```ini
   KIDS_VECTOR_BACKEND=keyword
   ```
3. 首次建议先跑 S5 或 `python run_ingest.py` 确保索引存在

## 目录结构
